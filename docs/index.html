Show code 
import json, glob, os
import pandas as pd
import matplotlib.pyplot as plt

# What we deliberately injected (the ground truth the diagnosis is checked against).
INJECTED = {
    "S1_clean":   "none (baseline)",
    "S2_adapter": "adapter read-through",
    "S3_lowqual": "low-quality 3' tails",
    "S4_overrep": "overrepresented/contaminant sequence",
}

rows = []
for f in sorted(glob.glob("results/*.fastp.json")):
    s = os.path.basename(f).replace(".fastp.json", "")
    d = json.load(open(f))
    bf, af = d["summary"]["before_filtering"], d["summary"]["after_filtering"]
    adapt = d.get("adapter_cutting", {}).get("adapter_trimmed_reads", 0)
    rows.append({
        "sample": s,
        "reads_before": bf["total_reads"],
        "reads_after": af["total_reads"],
        "q30_before": 100 * bf["q30_rate"],
        "q30_after": 100 * af["q30_rate"],
        "adapter_pct": 100 * adapt / max(1, bf["total_reads"]),
        "dup_pct": 100 * d.get("duplication", {}).get("rate", 0.0),
        "len_before": bf.get("read1_mean_length", 0),
        "len_after": af.get("read1_mean_length", 0),
        "reads_removed_pct": 100 * (1 - af["total_reads"] / max(1, bf["total_reads"])),
    })
df = pd.DataFrame(rows).set_index("sample")

1 · Diagnosis

Reading the per-sample QC, each sample’s dominant problem is flagged automatically from quantitative thresholds — then checked against what we actually injected.

Show code 
def diagnose(r):
    issues = []
    if r.adapter_pct > 15:           issues.append("adapter read-through")
    if r.q30_before < 80:            issues.append("low-quality 3' tails")
    if r.dup_pct > 30:               issues.append("overrepresented/contaminant sequence")
    return "; ".join(issues) if issues else "none (clean)"

diag = df.apply(diagnose, axis=1)
table = pd.DataFrame({
    "Injected (truth)": [INJECTED[s] for s in df.index],
    "Diagnosed from QC": diag.values,
    "Adapter %": df.adapter_pct.round(1).values,
    "Q30 before": df.q30_before.round(1).values,
    "Duplication %": df.dup_pct.round(1).values,
}, index=df.index)

3 · Verdict per sample

def verdict(s, r):
    if diag[s] == "none (clean)":
        return "Pass — usable as-is."
    return (f"Fixed — cleaned to {r.q30_after:.0f}% Q30, "
            f"{r.reads_after:,.0f} reads retained ({r.reads_removed_pct:.1f}% removed).")

  • S2_adapter (adapter read-through): Fixed — cleaned to 100% Q30, 80,000 reads retained (0.0% removed).
  • S3_lowqual (low-quality 3’ tails): Fixed — cleaned to 27% Q30, 31,568 reads retained (60.5% removed).

4 · Interpretation & caveats

NoteWhat this means — and where judgment matters

Every sample is now usable, with a documented reason for every read removed. The value here is not running fastp — anyone can do that — it is knowing what to look for, reading the QC correctly, and not over-trimming. Aggressive trimming can create bias (e.g. length-dependent mapping artifacts); the thresholds used here are deliberately conservative.

Caveats: (1) cross-species/contaminant screening (e.g. FastQ Screen, Kraken2) is the next step when an overrepresented sequence might be a different organism rather than a duplicate. (2) A suspected sample swap is diagnosed differently — via genotype/sex concordance, not read QC. (3) Synthetic data here makes the diagnosis checkable; real data needs the same disciplined reading.

What I’d do with your data: run this triage first, tell you plainly whether the data is salvageable, fix what’s fixable, and flag what isn’t — before you spend analysis time on a doomed run.
